# Product Checklist

- Cloudflare-first web application architecture is in place
- Next.js frontend scaffold is implemented
- Python backend API is implemented
- Fraud overview, anomaly detection, supplier review, expense review, refund monitoring, investigation, governance, and about sections are implemented
- Transaction, procurement, expense, refund, investigation, and governance agents are implemented
- CSV and text upload support is implemented
- Synthetic fraud data fallback is implemented
- Safe-use framing avoids legal conclusions and accusations
- Investigation artifacts, anomaly logs, and governance briefings are expected outputs
- Recurring scans and follow-up workflows are part of the specification
- Shared model-router assumptions are included
- Sandbox-ready heavy-task language is included for future batch fraud work
- Qwen 2.5 Instruct is the default for investigation copilots, executive summaries, and fraud explanations
- Mistral Small Instruct is the default for intake parsing, issue extraction, and routine fraud summaries
- bge-m3 is the default for fraud note retrieval, semantic search, and clustering
- Backend Docker packaging is included
- Cloudflare deployment guidance is included
